## Project Goal

Build a backend service that evaluates whether text-based creative work was likely written by a human or generated by AI, returns a confidence score, shows a transparent label to end users, supports appeals, and records every decision in an audit log.

## Core Requirements

- Content submission endpoint
- At least 2 distinct detection signals
- Confidence scoring with meaningful uncertainty
- Transparency labels for high-confidence human, high-confidence AI, and uncertain
- Appeals workflow
- Rate limiting
- Structured audit logging

## Detection Signals

### Signal 1: Groq LLM classification

- What it captures: semantic cues, coherence, style, and whether the writing feels human or AI-generated overall
- Why use it: strong holistic signal that can catch patterns heuristics miss

### Signal 2: Stylometric heuristics

- What it captures: sentence length variance, punctuation density, lexical diversity, repetition, and uniformity
- Why use it: cheap, deterministic, and complementary to the LLM signal

## Confidence Strategy

- Define how raw signal outputs become a final score
- Reserve an explicit uncertain range rather than forcing binary decisions
- Use a more conservative threshold for labeling content as AI-generated than for labeling it human-written, if needed

## Appeals Workflow

- Capture creator reasoning
- Log the appeal alongside the original decision
- Change the content status to `under review`

## Rate Limiting

- Decide a realistic per-user or per-IP limit for submission requests
- Document why the limit fits a writing platform and how it resists abuse

## Audit Log

- Store submission timestamp
- Store text hash or content identifier
- Store signals used
- Store confidence score
- Store transparency label
- Store appeals

## Architecture

```text
Client
  |
  v
Flask API
  |-- POST /submit  -> multi-signal detector -> confidence + label -> audit log
  |-- POST /appeal  -> appeal capture -> status update -> audit log
  |-- GET /log      -> structured audit history
```
