# Provenance Guard — planning.md

> Written before implementation. Update this document before starting any stretch feature.

---

## Project Goal

Build a backend service that evaluates whether text-based creative work was likely written by a human or generated by AI, returns a calibrated confidence score, shows a transparent label to end users, supports appeals from creators, and records every decision in a structured audit log.

---

## Architecture

### Narrative

When a creator submits text, the API validates the request, runs two independent detection signals (Groq LLM + stylometric heuristics), combines them into a single AI-likelihood score, maps that score to one of three attribution categories using asymmetric thresholds, generates the matching transparency label text, persists a structured audit-log entry, and returns the result. If the creator disagrees, they submit an appeal with reasoning; the system updates the content status to `under_review`, appends appeal metadata to the same audit record, and returns confirmation. A reviewer (out of scope for automation) would later resolve the appeal manually.

### Submission flow

```text
Client
  |
  |  POST /submit { text, creator_id }
  v
Flask API (rate limit check)
  |
  |-- validate JSON body
  |
  v
content_id = UUID
text_hash = SHA-256(text)
  |
  +--> Signal 1: Groq LLM classifier
  |         |
  |         +--> llm_score (0.0-1.0, higher = more AI-like)
  |
  +--> Signal 2: Stylometric heuristics
  |         |
  |         +--> heuristic_score (0.0-1.0, higher = more AI-like)
  |
  v
confidence = combine(llm_score, heuristic_score)
attribution, label_text = map_to_label(confidence)
  |
  v
Audit log write (JSON/SQLite)
  |
  v
Response { content_id, attribution, confidence, label, llm_score, heuristic_score, status }
```

### Appeal flow

```text
Client
  |
  |  POST /appeal { content_id, creator_reasoning }
  v
Flask API
  |
  |-- lookup original submission by content_id
  |-- reject if content_id unknown
  |
  v
Update stored record:
  status = "under_review"
  appeal_reasoning = creator_reasoning
  appeal_timestamp = ISO-8601 UTC
  |
  v
Append appeal fields to audit log entry
  |
  v
Response { content_id, status: "under_review", message }
```

### API surface

| Endpoint | Method | Request | Response |
|----------|--------|---------|----------|
| `/health` | GET | — | `{ "status": "ok" }` |
| `/submit` | POST | `{ "text": string, "creator_id": string }` | `{ "content_id", "attribution", "confidence", "label", "llm_score", "heuristic_score", "status" }` |
| `/appeal` | POST | `{ "content_id": string, "creator_reasoning": string }` | `{ "content_id", "status", "message" }` |
| `/log` | GET | — | `{ "entries": [ ... ] }` |

**Field definitions**

- `attribution`: one of `likely_ai`, `likely_human`, `uncertain`
- `confidence`: float in `[0.0, 1.0]` representing AI-likelihood (higher = more likely AI-generated)
- `label`: exact user-facing transparency label string
- `status`: `classified` after submission; `under_review` after appeal

## Detection Signals

### Signal 1: Groq LLM classification

**What it measures:** Semantic and stylistic coherence, generic phrasing, hedging patterns, and overall “AI voice” vs. casual human voice.

**Output format:** `llm_score: float` in `[0.0, 1.0]`, where `1.0` means the model is highly confident the text is AI-generated.

**How it is produced:**

- Model: `llama-3.3-70b-versatile`
- Prompt asks the model to assess whether the text reads as human-written or AI-generated
- Model must return only a JSON object: `{ "ai_likelihood": 0.0-1.0 }`
- Parse and clamp to `[0.0, 1.0]`
- On API failure, fall back to `0.5` and mark the run as uncertain-friendly (do not auto-accuse)

**Why this signal:** It catches holistic patterns such as polished essay tone, template transitions (“Furthermore…”, “It is important to note…”), and unnaturally balanced structure that heuristics miss.

**Blind spots:**

- Formal human academic or legal writing can look “AI-polished”
- Short casual human posts may look ambiguous
- Lightly edited AI text can score mid-range
- Non-native English writers may be misread as generated if the model overweights “formal” as “AI”

### Signal 2: Stylometric heuristics

**What it measures:** Structural uniformity of human vs. AI writing — sentence-length variability, lexical diversity, and punctuation regularity.

**Output format:** `heuristic_score: float` in `[0.0, 1.0]`, where higher means more AI-like structure.

**Metrics computed in pure Python:**

1. **Sentence-length coefficient of variation (CV)**  
   - Split text on `.!?` into sentences  
   - CV = `stdev(lengths) / mean(lengths)`  
   - AI text often has low variance → low CV → higher AI sub-score

2. **Type-token ratio (TTR)**  
   - Unique words / total words (lowercased, alphabetic tokens only)  
   - Very low TTR suggests repetitive or templated phrasing

3. **Transition-phrase density**  
   - Count phrases such as `furthermore`, `in conclusion`, `it is important to note`, `additionally`  
   - Normalize by sentence count

**Sub-score mapping (each normalized to 0.0-1.0, higher = more AI-like):**

- `cv_subscore = clamp(1.0 - (cv / 0.45), 0, 1)`  
  - CV below ~0.20 looks unusually uniform
- `ttr_subscore = clamp(1.0 - (ttr / 0.55), 0, 1)`  
  - TTR below ~0.35 looks repetitive
- `transition_subscore = clamp(transition_count / max(sentence_count, 1) / 0.35, 0, 1)`

**Final heuristic score:**

```text
heuristic_score = 0.45 * cv_subscore + 0.35 * ttr_subscore + 0.20 * transition_subscore
```

**Why this signal:** Cheap, deterministic, and independent of the LLM. It captures structural regularity that differs from semantic judgment.

**Blind spots:**

- Poetry with repeated lines or simple vocabulary
- Deliberately minimalist human writing
- Lists, bullet notes, or chat-style fragments with very short sentences
- Heavily edited AI text that a human manually roughed up may look more “variable” and score lower

### Why these two signals together

They are genuinely distinct: one is semantic/holistic, one is structural/statistical. Agreement increases confidence; disagreement pushes results toward the uncertain band instead of a harsh accusation.

## Uncertainty Representation

### What the confidence score means

`confidence` is an **AI-likelihood score**, not “confidence that we are right about everything.”

- `0.05` → strong evidence of human writing
- `0.50` → mixed or weak evidence; system should not accuse
- `0.92` → strong evidence of AI-generated text

This design avoids treating `0.51` and `0.95` as the same kind of answer.

### Combining signals

```text
confidence = (0.65 * llm_score) + (0.35 * heuristic_score)
```

LLM weight is higher because it captures meaning and tone; heuristics act as a corrective when structure is suspicious or unusually uniform.

If the two signals disagree by more than `0.35` (`abs(llm_score - heuristic_score) > 0.35`), cap the final score toward the uncertain middle:

```text
if abs(llm_score - heuristic_score) > 0.35:
    confidence = 0.5 * confidence + 0.25
```

This prevents a single overconfident signal from forcing a high-confidence accusation.

### Asymmetric label thresholds

False positives (calling human work AI) are worse than false negatives on a writing platform. Therefore:

| Attribution | Threshold on `confidence` | Rationale |
|-------------|---------------------------|-----------|
| `likely_human` | `confidence <= 0.30` | Only call something human-written when both signals are fairly low |
| `uncertain` | `0.30 < confidence < 0.75` | Mixed or weak evidence → do not accuse |
| `likely_ai` | `confidence >= 0.75` | Require stronger evidence before showing an AI attribution label |

The AI threshold (`0.75`) is farther from center than the human threshold (`0.30`), so the system is harder to trigger into accusing a creator of using AI.

### False-positive walkthrough

**Scenario:** A human poet submits a short piece with repeated phrases and uniform line lengths.

1. Heuristics may score mid-high because of low sentence-length variance and repetition.
2. LLM may also score moderately high because the poem sounds polished or patterned.
3. Combined score lands in the uncertain band rather than `likely_ai` unless both signals are very high.
4. Label shows the uncertain variant, not an accusation.
5. Creator submits an appeal explaining the piece is original poetry with intentional repetition.
6. Status becomes `under_review`; appeal reasoning is logged next to the original scores for a human reviewer.

## Transparency Label Design

These are the exact user-facing strings returned in the API `label` field and shown on the platform.

### High-confidence AI (`likely_ai`, `confidence >= 0.75`)

```text
Likely AI-generated (high confidence). Our analysis found strong signals associated with generated writing. If you wrote this yourself, you can appeal this result.
```

### High-confidence human (`likely_human`, `confidence <= 0.30`)

```text
Likely human-written (high confidence). We did not find strong AI-generation signals in this submission.
```

### Uncertain (`0.30 < confidence < 0.75`)

```text
Origin uncertain. Our checks found mixed signals, so we are not making a strong attribution claim. You can still appeal if you want a human review.
```

## Appeals Workflow

### Who can appeal

Any creator who received a classification for their own submission. For this project, appeals are accepted whenever a valid `content_id` exists; no auth layer is required beyond knowing the ID returned by `/submit`.

### Request

```json
POST /appeal
{
  "content_id": "3f7a2b1e-...",
  "creator_reasoning": "I wrote this from personal experience. I am a non-native English speaker and my writing may look more formal than typical casual posts."
}
```

### System behavior

1. Look up the original submission by `content_id`
2. Return `404` if not found
3. Update stored record:
   - `status = "under_review"`
   - `appeal_reasoning = creator_reasoning`
   - `appeal_timestamp = UTC ISO-8601`
4. Append/update the audit-log entry with appeal metadata
5. Do **not** automatically re-run detection in v1

### Response

```json
{
  "content_id": "3f7a2b1e-...",
  "status": "under_review",
  "message": "Appeal received. This submission is now under review."
}
```

### What a human reviewer would see

- Original text hash / content ID
- Original attribution, confidence, label
- `llm_score`, `heuristic_score`
- Creator ID
- Appeal reasoning and timestamp
- Prior audit history for that content ID

## Rate Limiting

**Chosen limits on `POST /submit`:**

```text
10 per minute; 100 per day
```

Applied per remote IP using Flask-Limiter with `storage_uri="memory://"`.

**Reasoning:**

- A real creator might submit a draft, revise it, and resubmit a few times — roughly 3-5 attempts in a short session is normal.
- `10/minute` allows normal iterative writing without blocking legitimate use.
- `100/day` prevents scripted flooding or brute-force probing of the detector while still allowing heavy legitimate same-day editing.
- Appeals are not rate-limited as aggressively in v1 because they should be rare and low-volume relative to submissions.

## Anticipated Edge Cases

### 1. Formal human academic writing

A human-written policy memo or literature review may use balanced structure, transition phrases, and stable sentence lengths. Heuristics may score it as AI-like, and the LLM may also rate it as polished/generated. Expected behavior: combined score often lands in the uncertain band unless both signals are very high; the label should avoid a careless AI accusation.

### 2. Poetry or lyrics with repetition

Repeated lines, simple vocabulary, and uniform line lengths can reduce sentence-length variance and TTR. Heuristics may overestimate AI-likelihood even when a human intentionally wrote the piece. Expected behavior: disagreement dampening and uncertain labeling protect against false positives; appeals are especially important here.

### 3. Lightly edited AI drafts

A user pastes AI output, then manually shortens sentences or swaps a few words. Signals may diverge — heuristics lower, LLM still mid-high. Expected behavior: disagreement rule pulls the score toward uncertain rather than a confident AI label.

### 4. Very short submissions

Texts under ~40 characters may produce unstable heuristic metrics. Expected behavior: if sentence count `< 2`, reduce heuristic weight to `0.15` and rely more on LLM score, but default toward uncertain if the LLM score is not extreme.

## Validation Plan

Use these four inputs during Milestone 4 testing:

1. **Clearly AI-generated** — polished essay with template transitions → expect `confidence >= 0.75`
2. **Clearly human casual** — lowercase chatty restaurant review → expect `confidence <= 0.30`
3. **Borderline formal human** — academic paragraph about monetary policy → expect uncertain band
4. **Lightly edited AI** — remote-work essay with mild human edits → expect uncertain or mid-range, not extreme human
